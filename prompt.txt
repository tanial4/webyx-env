openenv-cli create a real-world RL environment for benchmarking 
AI agents on web accessibility auditing. The environment is called 
webyx-env and simulates the task a QA engineer does daily: auditing 
and fixing HTML pages for WCAG compliance.

OBSERVATION:
- Current HTML snippet being audited
- List of detected violations with WCAG level (A, AA or AAA), 
  element selector, and description
- Number of violations remaining per level
- Current step number and max steps

ACTION:
- Type: one of detect / fix / skip
- Target: CSS selector of the element to act on
- Proposed fix: the corrected HTML attribute or element

REWARD:
- Positive signal for each violation correctly resolved in a step
- Partial credit for detecting violations even without fixing them
- Penalty for introducing new violations into the HTML
- Penalty for skipping a level-A critical violation
- Zero reward for invalid HTML output

THREE TASKS:
- Task easy: fix missing alt attributes on images in a simple HTML 
  page. 3 to 5 images without alt text. Grader scores percentage 
  of alt attributes correctly added and non-empty.
- Task medium: fix multiple level-AA violations in a form: missing 
  labels, inputs without ARIA roles, low contrast text. Grader 
  scores detection accuracy and correctness of each fix 
  independently.
- Task hard: full audit of a complex page mixing level A, AA and 
  AAA violations, some interdependent. Agent must prioritize by 
  severity, fix without breaking valid existing attributes. Grader 
  runs axe-core logic on the resulting HTML and scores residual 
  violations weighted by severity level.

Episode ends when all violations are resolved or max steps reached.
Each grader must return a deterministic score between 0.0 and 1.0.
